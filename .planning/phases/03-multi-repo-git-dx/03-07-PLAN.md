<objective>
Disable external repo git hooks during sync operations to prevent false failures.

Purpose: UAT Test 4 revealed that synced Nx repos with Husky post-checkout hooks cause exit code 126 during sync because the hooks run in a broken context (wrong node path, missing deps). The sync executor should disable hooks by default for all git operations on external repos, with an opt-out per repo.

Output: Updated git commands, config schema, sync executor, and passing tests.
</objective>

  <name>Task 1: Add disableHooks parameter to git commands and config schema</name>
  <files>packages/op-nx-polyrepo/src/lib/git/commands.ts, packages/op-nx-polyrepo/src/lib/config/schema.ts</files>
  <behavior>
    - execGit prepends ['-c', 'core.hooksPath=__op-nx_polyrepo_disable-hooks__'] to args when disableHooks is true
    - execGit does NOT prepend hooks args when disableHooks is false or undefined
    - gitClone accepts disableHooks in CloneOptions and passes to execGit
    - gitPull, gitFetch, gitPullRebase, gitPullFfOnly accept optional disableHooks parameter
    - gitFetchTag accepts optional disableHooks parameter and applies to both fetch and checkout calls
    - NormalizedRepoEntry remote type includes optional disableHooks boolean (defaults to true)
    - Zod schema for remote repo object includes disableHooks: z.boolean().default(true)
    - NormalizedRepoEntry local type is unchanged (local repos run hooks normally)
  </behavior>
  <action>
**In `git/commands.ts`:**

1. Modify `execGit` signature to accept a third optional `disableHooks?: boolean` parameter.
2. When `disableHooks` is true, prepend `['-c', 'core.hooksPath=__op-nx_polyrepo_disable-hooks__']` before the existing args array. This tells git to look for hooks in a nonexistent directory, effectively disabling all hooks.
3. Update `CloneOptions` to include `disableHooks?: boolean`.
4. Update `gitClone` to pass `options.disableHooks` to `execGit`.
5. Update `gitPull`, `gitFetch`, `gitPullRebase`, `gitPullFfOnly` signatures to accept `(cwd: string, disableHooks?: boolean)` and pass it to `execGit`.
6. Update `gitFetchTag` signature to accept `(cwd: string, tag: string, depth?: number, disableHooks?: boolean)` and pass to both `execGit` calls.

**In `config/schema.ts`:**

7. Add `disableHooks: z.boolean().default(true)` to the `remoteRepoObject` zod schema (before `.strict()`).
8. Update the `NormalizedRepoEntry` remote type union member to include `disableHooks: boolean`.
9. In `normalizeRepos`, when building remote entries: for string entries, set `disableHooks: true` (default). For object entries, set `disableHooks: entry.disableHooks ?? true` (the zod default handles this, but be explicit in the normalizer for clarity).

All changes are backward-compatible: callers that do not pass `disableHooks` get existing behavior (hooks enabled).
