  <name>Task 1: Add getWorkingTreeState and getAheadBehind to detect.ts</name>
  <files>packages/op-nx-polyrepo/src/lib/git/detect.ts, packages/op-nx-polyrepo/src/lib/git/detect.spec.ts</files>
  <behavior>
    - getWorkingTreeState: parses `git status --porcelain=v1` output into { modified, staged, deleted, untracked, conflicts } counts
    - getWorkingTreeState: counts `??` lines as untracked
    - getWorkingTreeState: counts lines where Y='M' as modified (working tree changes)
    - getWorkingTreeState: counts lines where X in [M,A,D,R,C] as staged
    - getWorkingTreeState: counts lines where X='D' or Y='D' as deleted (but not double-counted if same line)
    - getWorkingTreeState: counts conflict patterns (UU, AA, DD, AU, UA, DU, UD) as conflicts
    - getWorkingTreeState: a file with MM (both staged and modified) increments both staged and modified counts -- this is correct per user decision
    - getWorkingTreeState: empty porcelain output returns all zeros
    - getAheadBehind: parses `git rev-list --left-right --count HEAD...@{u}` output "2\t3" into { ahead: 2, behind: 3 }
    - getAheadBehind: returns null when command fails (detached HEAD, no upstream, shallow clone)
    - getAheadBehind: returns { ahead: 0, behind: 0 } for "0\t0" output
  </behavior>
  <action>
    Export two new interfaces and two new async functions from detect.ts:

    1. `WorkingTreeState` interface: `{ modified: number; staged: number; deleted: number; untracked: number; conflicts: number }`
    2. `AheadBehind` interface: `{ ahead: number; behind: number }`
    3. `getWorkingTreeState(cwd: string): Promise<WorkingTreeState>` -- calls `execGitOutput(['status', '--porcelain=v1'], cwd)`, parses each line by XY characters. Logic:
       - Check conflict patterns first (X='U' || Y='U' || (X='A' && Y='A') || (X='D' && Y='D')) -> conflicts++, continue
       - Check untracked (X='?' && Y='?') -> untracked++, continue
       - If X in 'MADRC' -> staged++
       - If Y='M' -> modified++
       - If Y='D' -> deleted++ (else if X='D' -> deleted++)
       Note: staged and modified/deleted are NOT mutually exclusive for a single line. A file with `MM` increments both staged (X='M') and modified (Y='M').
    4. `getAheadBehind(cwd: string): Promise<AheadBehind | null>` -- calls `execGitOutput(['rev-list', '--left-right', '--count', 'HEAD...@{u}'], cwd)`, parses "ahead\tbehind", returns null on any error (try/catch).

    Keep all existing exports unchanged. The existing `execGitOutput` private helper is reused by both new functions.

    Add tests in detect.spec.ts following the existing `setupExecFileMock` pattern. For getWorkingTreeState, mock execFile to return various porcelain outputs. For getAheadBehind, mock both success and failure scenarios.
  </action>
  <verify>
    <automated>npm exec nx test @op-nx/polyrepo -- --reporter=verbose -t "getWorkingTreeState|getAheadBehind"</automated>
  </verify>
  <done>getWorkingTreeState correctly parses all porcelain status categories. getAheadBehind returns counts on success and null on failure. All existing detect.spec.ts tests still pass.</done>
