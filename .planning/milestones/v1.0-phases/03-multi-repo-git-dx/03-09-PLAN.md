<objective>
Make sync executor only install dependencies when HEAD actually changed during the sync operation.

Purpose: Currently, `tryInstallDeps` is called unconditionally after every sync strategy (pull, fetch, tag fetch, etc.), which wastes time re-installing deps when the repo is already at the target state. Per user report: "sync installs deps after every tag fetch even when repo is already at that tag."

Output: Conditional install logic comparing HEAD SHA before and after sync, with install only when HEAD moved.
</objective>
