  <name>Task 2: Harden TSConfig and fix vitest.config.mts</name>
  <files>tsconfig.base.json, packages/op-nx-polyrepo/vitest.config.mts</files>
  <action>
1. Add three flags to `tsconfig.base.json` `compilerOptions`:
   - `"noUncheckedIndexedAccess": true` -- Record and array indexing returns `T | undefined`
   - `"exactOptionalPropertyTypes": true` -- distinguishes "missing" from "undefined"
   - `"noPropertyAccessFromIndexSignature": true` -- forces bracket notation for index signatures

   Per Pitfall 4: `exactOptionalPropertyTypes` may conflict with @nx/devkit types. If typecheck fails on Nx type usage, REMOVE `exactOptionalPropertyTypes` only (keep the other two). Document the removal reason in a code comment.

2. Fix `packages/op-nx-polyrepo/vitest.config.mts`:
   - The `provider: 'v8' as const` line uses an `as` assertion. With `allowAsConst: true` in the ESLint config (Task 1), this is now permitted. However, since `'v8'` is already a string literal, the `as const` is redundant -- the type is already narrowed. Remove `as const` entirely: just use `provider: 'v8'`. If the type system needs the literal type, use `satisfies` instead.
