    commands.spec.ts (mock execFile to verify arg construction):
    - gitClone constructs ['clone', '--depth', '1', url, targetDir] for default shallow clone
    - gitClone constructs ['clone', url, targetDir] when depth is 0 (full clone)
    - gitClone constructs ['clone', '--depth', '1', '--branch', ref, url, targetDir] when ref is specified
    - gitPull constructs ['pull'] with cwd set to repo directory
    - gitFetch constructs ['fetch'] with cwd set to repo directory
    - gitFetchTag constructs ['fetch', '--depth', '1', 'origin', 'tag', tagName] then ['checkout', tagName]
    - gitRebase constructs ['pull', '--rebase'] with cwd
    - gitFfOnly constructs ['pull', '--ff-only'] with cwd

    detect.spec.ts:
    - isGitUrl returns true for git@, https://, ssh://, file:// URLs
    - isGitUrl returns false for local paths (D:/projects/repo, ../repo, /home/user/repo)
    - detectRepoState returns 'cloned' when .repos/<alias>/.git exists
    - detectRepoState returns 'not-synced' when .repos/<alias> does not exist
    - detectRepoState returns 'referenced' for local path repos that exist
    - getCurrentBranch returns branch name from git rev-parse
    - getCurrentRef returns tag/commit when HEAD is detached

  </behavior>
  <action>
    Create `packages/nx-openpolyrepo/src/lib/git/commands.ts`:

    1. Import `execFile` from `node:child_process` and promisify it.

    2. Implement thin wrappers. Each function constructs the correct args array and calls `execFile('git', args, { cwd })`. Handle stderr correctly: git writes progress/info to stderr even on success, so only treat non-zero exit code (rejection) as failure.

    3. Functions to implement:
       - `gitClone(url, targetDir, { depth?, ref? })` -- constructs clone command. Depth defaults to 1. If depth is 0, omit --depth flag entirely. If ref specified, add `--branch <ref>`.
       - `gitPull(cwd)` -- simple `git pull`
       - `gitFetch(cwd)` -- simple `git fetch`
       - `gitPullRebase(cwd)` -- `git pull --rebase`
       - `gitPullFfOnly(cwd)` -- `git pull --ff-only`
       - `gitFetchTag(cwd, tag, depth?)` -- `git fetch --depth=<depth> origin tag <tag>` then `git checkout <tag>`

    4. IMPORTANT: Normalize path separators to forward slashes when passing paths as git arguments on Windows. Use a helper: `const gitPath = (p: string) => p.replace(/\\/g, '/')`.

    Create `packages/nx-openpolyrepo/src/lib/git/detect.ts`:

    5. `isGitUrl(value: string): boolean` -- tests against `/^(git@|https?:\/\/|ssh:\/\/|file:\/\/)/`.

    6. `detectRepoState(alias, entry: NormalizedRepoEntry, workspaceRoot)` -- checks filesystem:
       - For remote: checks `.repos/<alias>/.git` exists -> 'cloned' or 'not-synced'
       - For local: checks `entry.path` exists -> 'referenced' or 'not-synced'

    7. `getCurrentBranch(cwd): Promise<string | null>` -- runs `git rev-parse --abbrev-ref HEAD`. Returns null if detached.

    8. `getCurrentRef(cwd): Promise<string>` -- runs `git describe --tags --exact-match HEAD` (tag) or falls back to `git rev-parse --short HEAD` (commit SHA).

    Write tests FIRST (red), then implement (green). For commands.spec.ts, mock `node:child_process` execFile to capture args without actually running git. For detect.spec.ts, mock `node:fs/promises` stat.

<task type="auto" tdd="true">
  <name>Task 2: Implement polyrepo-sync executor</name>
  <files>
    packages/nx-openpolyrepo/src/lib/executors/sync/executor.ts,
    packages/nx-openpolyrepo/src/lib/executors/sync/schema.json,
    packages/nx-openpolyrepo/src/lib/executors/sync/executor.spec.ts
  </files>
  <behavior>
    executor.spec.ts (mock git commands and config reading):
    - clones remote repo when .repos/<alias> does not exist
    - pulls remote repo when .repos/<alias> already exists and ref is a branch
    - re-fetches tag when .repos/<alias> already exists and ref looks like a tag
    - pulls local path repo when it is a git repo
    - skips local path repo pull when path does not exist (warns)
    - uses configured depth for clone (depth:0 = full, depth:1 = shallow)
    - uses configured ref as --branch during clone
    - processes all repos in parallel (Promise.allSettled)
    - returns { success: true } when all repos succeed
    - returns { success: false } when any repo fails
    - logs per-repo results (cloning/pulling/done/failed)
    - logs summary at end (N synced, M failed)
    - respects strategy option: 'pull' (default), 'fetch', 'rebase', 'ff-only'
    - strategy 'fetch' calls gitFetch instead of gitPull
    - strategy 'rebase' calls gitPullRebase
    - strategy 'ff-only' calls gitPullFfOnly
  </behavior>
  <action>
    Create `packages/nx-openpolyrepo/src/lib/executors/sync/schema.json`:
    ```json
    {
      "$schema": "https://json-schema.org/schema",
      "type": "object",
      "properties": {
        "strategy": {
          "type": "string",
          "enum": ["fetch", "pull", "rebase", "ff-only"],
          "default": "pull",
          "description": "Git update strategy for existing repos"
        }
      },
      "additionalProperties": false
    }
    ```

    Create `packages/nx-openpolyrepo/src/lib/executors/sync/executor.ts`:

    1. Define `SyncExecutorOptions` interface: `{ strategy?: 'fetch' | 'pull' | 'rebase' | 'ff-only' }`.

    2. Read plugin config from nx.json inside the executor (use `readNxJson` from `@nx/devkit`). Find the plugin entry matching `'nx-openpolyrepo'`, validate with `validateConfig`, normalize with `normalizeRepos`.

    3. For each normalized repo entry, create an async function that:
       - For remote repos:
         - If NOT cloned (`.repos/<alias>/.git` doesn't exist): call `gitClone(entry.url, path.join(root, '.repos', entry.alias), { depth: entry.depth, ref: entry.ref })`
         - If cloned AND entry.ref matches a tag pattern (starts with 'v' and contains dots, OR any ref that was originally a tag): call `gitFetchTag(repoPath, entry.ref, entry.depth)`
         - If cloned AND branch (or no ref): call the appropriate strategy function (`gitPull` for 'pull', `gitFetch` for 'fetch', `gitPullRebase` for 'rebase', `gitPullFfOnly` for 'ff-only')
       - For local repos:
         - Check if path exists and is a git repo (.git directory)
         - If yes: apply strategy (pull/fetch/rebase/ff-only)
         - If path doesn't exist: warn and skip

    4. Run all repo sync functions with `Promise.allSettled()` for parallel execution.

    5. Collect results. Log per-repo outcome using `logger.info` (success) or `logger.error` (failure). Log summary: "Summary: N synced, M failed".

    6. Return `{ success: !hasFailures }`.

    7. Export the executor as default: `export default syncExecutor`.

    Write tests FIRST. Mock `readNxJson`, git command functions, and filesystem checks. Verify the executor calls the right git commands with right args for each scenario.
