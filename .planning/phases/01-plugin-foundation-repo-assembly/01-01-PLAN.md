  <behavior>
    schema.spec.ts:
    - accepts string URL (git@github.com:org/repo.git) as repo entry
    - accepts string URL (https://github.com/org/repo.git) as repo entry
    - accepts string URL (ssh://git@github.com/org/repo.git) as repo entry
    - accepts string URL (file:///path/to/repo) as repo entry
    - accepts string local path (D:/projects/repo-b) as repo entry
    - accepts string local path (../relative/repo) as repo entry
    - accepts object with url field for remote repo
    - accepts object with path field for local repo
    - accepts object with url + ref + depth options
    - accepts depth: 0 (full clone)
    - rejects empty string
    - rejects empty repos map
    - rejects object with both url and path (ambiguous)
    - rejects object with neither url nor path
    - rejects negative depth
    - rejects non-integer depth
    - rejects missing repos key entirely

    validate.spec.ts:
    - validateConfig returns parsed config for valid input
    - validateConfig throws with zod error details for invalid input
    - normalizeRepos converts string URL to { type: 'remote', url, alias, ref: undefined, depth: 1 }
    - normalizeRepos converts string path to { type: 'local', path, alias }
    - normalizeRepos converts object URL to { type: 'remote', url, alias, ref, depth }
    - normalizeRepos converts object path to { type: 'local', path, alias }
    - normalizeRepos defaults depth to 1 for remote repos
    - warnIfReposNotGitignored warns when .repos/ is not in .gitignore (mock fs)
    - warnIfReposNotGitignored does not warn when .repos/ is in .gitignore (mock fs)
    - warnUnsyncedRepos warns for remote repos missing from .repos/ directory (mock fs)
    - warnUnsyncedRepos does not warn for local path repos
  </behavior>
  <action>
    Create the config schema in `packages/nx-openpolyrepo/src/lib/config/schema.ts`:

    1. Define the git URL pattern regex: `/^(git@|https?:\/\/|ssh:\/\/|file:\/\/)/`

    2. Define zod schemas per CONTEXT.md config shape:
       - `remoteRepoObject`: `z.object({ url: z.string().regex(gitUrlPattern), ref: z.string().optional(), depth: z.number().int().min(0).optional() })`
       - `localRepoObject`: `z.object({ path: z.string().min(1) })`
       - `repoEntry`: `z.union([z.string().min(1), remoteRepoObject, localRepoObject])`
       - `polyrepoConfigSchema`: `z.object({ repos: z.record(z.string().min(1), repoEntry) })`
       - Export inferred type `PolyrepoConfig`

    3. Define `NormalizedRepoEntry` type:
       ```typescript
       export type NormalizedRepoEntry =
         | { type: 'remote'; alias: string; url: string; ref?: string; depth: number }
         | { type: 'local'; alias: string; path: string };
       ```

    4. Implement `normalizeRepos(config: PolyrepoConfig): NormalizedRepoEntry[]` that converts config entries:
       - String matching gitUrlPattern -> `{ type: 'remote', alias: key, url: value, depth: 1 }`
       - String not matching -> `{ type: 'local', alias: key, path: value }`
       - Object with `url` -> `{ type: 'remote', alias: key, url, ref, depth: depth ?? 1 }`
       - Object with `path` -> `{ type: 'local', alias: key, path }`

    Create validation utilities in `packages/nx-openpolyrepo/src/lib/config/validate.ts`:

    5. `validateConfig(options: unknown): PolyrepoConfig` -- calls `polyrepoConfigSchema.safeParse(options)`, throws with formatted zod error on failure, returns `result.data` on success.

    6. `warnIfReposNotGitignored(workspaceRoot: string): Promise<void>` -- reads `.gitignore`, checks for `.repos` or `.repos/` entry, warns via `logger` if missing.

    7. `warnUnsyncedRepos(config: PolyrepoConfig, workspaceRoot: string): void` -- checks if `.repos/<alias>/` exists for each remote repo entry, logs warning for missing ones suggesting `nx polyrepo-sync`.

    Write tests FIRST (red), then implement (green).
  </action>

    <automated>npx nx test nx-openpolyrepo -- --reporter=verbose 2>&1</automated>
  </verify>
  <done>Schema accepts all valid config forms from CONTEXT.md, rejects invalid forms with clear errors, normalizeRepos produces typed entries, validation utilities work correctly. All tests pass.</done>
