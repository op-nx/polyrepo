<objective>
Graph extraction pipeline: extract graph JSON from synced repos, cache with two-layer invalidation, and transform with namespace prefixing and target rewriting.

Purpose: This is the core graph pipeline that makes external repo projects discoverable. Extraction shells out to `nx graph --print` in each repo. Caching ensures this ~2-4s per repo cost is only paid when repos change. Transformation rewrites project names, targets, tags, and paths for the host workspace.

Output: extract.ts, cache.ts, transform.ts -- the complete graph pipeline ready to be wired into plugin hooks (Plan 03).
</objective>

    **`computeOuterHash(config, workspaceRoot, optionsHash): Promise<string>`** (not exported, internal):

      - Use `Promise.all` to call `extractGraphFromRepo(repoPath)` for each.

    - Return `graphReport`.

    **Cache restoration (cold start):**

    **Important:** Follow @nx/gradle's defensive pattern -- `populateGraphReport` is called from both `createNodesV2` and `createDependencies`. The module-level check ensures only one extraction per hash.

  <behavior>
    transformGraphForRepo tests:
    - Prefixes project names with `repoAlias/` separator: `my-lib` -> `repo-b/my-lib`
    - Rewrites project root to `.repos/<alias>/<original-root>`: `libs/my-lib` -> `.repos/repo-b/libs/my-lib`
    - Rewrites sourceRoot similarly: `libs/my-lib/src` -> `.repos/repo-b/libs/my-lib/src`
    - Handles missing sourceRoot (undefined stays undefined)
    - Preserves projectType from node data (or falls back to node.type)
    - Preserves existing tags and appends `polyrepo:external` + `polyrepo:<repo-alias>`
    - Preserves metadata from original node
    - Rewrites each target to use `executor: '@op-nx/polyrepo:run'` with options `{ repoAlias, originalProject, targetName }`
    - Copies inputs from original target to proxy target
    - Copies outputs from original target to proxy target (output paths using `{projectRoot}` tokens work as-is because Nx resolves them against the new root)
    - Copies cache, parallelism, metadata from original target
    - Rewrites dependsOn: literal project references get prefixed, caret syntax (`^build`) passes through unchanged, self-references (`build`) pass through unchanged
    - Copies configurations from original target
    - Handles project with no targets (empty targets object)
    - Handles project with no tags (empty array, still gets auto-tags)
    - Transforms dependencies: both source and target get prefixed with `repoAlias/`
    - Dependencies use string type value (passed through from external graph)
    - Handles project with no dependencies (empty array)
    - All paths use forward slashes (no backslashes on Windows)
  </behavior>
  <action>
    **Create `packages/op-nx-polyrepo/src/lib/graph/transform.ts`:**

    Implement `transformGraphForRepo(repoAlias: string, rawGraph: ExternalGraphJson, workspaceRoot: string): { nodes: Record<string, TransformedNode>, dependencies: Array<{ source: string, target: string, type: string }> }`:

    Import types from `./types`. Import `TargetConfiguration` from `@nx/devkit`.

    **Project node transformation** (iterate `rawGraph.graph.nodes`):
    For each `[originalName, node]`:
    - `namespacedName = \`\${repoAlias}/\${originalName}\``
    - `hostRoot = \`.repos/\${repoAlias}/\${node.data.root}\`` (use forward slashes always)
    - `hostSourceRoot = node.data.sourceRoot ? \`.repos/\${repoAlias}/\${node.data.sourceRoot}\` : undefined`

    **Target rewriting** (iterate `node.data.targets ?? {}`):
    For each `[targetName, targetConfig]`:
    ```
    proxyTarget = {
      executor: '@op-nx/polyrepo:run',
      options: { repoAlias, originalProject: originalName, targetName },
      inputs: targetConfig.inputs,
      outputs: targetConfig.outputs,
      cache: targetConfig.cache,
      dependsOn: rewriteDependsOn(targetConfig.dependsOn, repoAlias),
      configurations: targetConfig.configurations,
      parallelism: targetConfig.parallelism,
      metadata: targetConfig.metadata,
    }
    ```

    **`rewriteDependsOn(dependsOn, repoAlias)`** helper:
    - If undefined or empty, return undefined.
    - For each entry:
      - If string starting with `^` (caret syntax like `^build`): pass through unchanged (Nx resolves against dependency graph).
      - If plain string (like `build`): pass through unchanged (self-reference).
      - If object with `projects` field containing specific project names: prefix each project name with `repoAlias/`.
      - If object with `target` but no `projects`: pass through (relative reference).

    **Tag injection:**
    ```
    tags = [...(node.data.tags ?? []), 'polyrepo:external', \`polyrepo:\${repoAlias}\`]
    ```

    **Build TransformedNode:**
    ```
    {
      name: namespacedName,
      root: hostRoot,
      projectType: node.data.projectType ?? node.type,
      sourceRoot: hostSourceRoot,
      targets: proxyTargets,
      tags,
      metadata: node.data.metadata,
    }
    ```

    **Dependency transformation** (iterate `rawGraph.graph.dependencies`):
    For each `[sourceName, deps]`, for each `dep`:
    ```
    { source: \`\${repoAlias}/\${dep.source}\`, target: \`\${repoAlias}/\${dep.target}\`, type: dep.type }
    ```

    **Path normalization:** Ensure ALL paths use forward slashes. Use a helper `normalizePath(p: string): string => p.replace(/\\\\/g, '/')` or import from `@nx/devkit` if available (check `normalizePath` export).

    **Write comprehensive tests** with fixture data representing a typical external graph (2-3 projects with targets, dependencies, tags). Test edge cases: no targets, no tags, no dependencies, no sourceRoot, dependsOn with caret syntax.
