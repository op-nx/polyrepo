<objective>
Fix stdout contamination bug in extractGraphFromRepo that prevents external projects from appearing in `nx show projects`.

Purpose: The child `nx graph --print` subprocess inherits `NX_VERBOSE_LOGGING` from the parent Nx daemon environment. This causes `[isolated-plugin]` diagnostic lines on stdout before the JSON payload, making `JSON.parse(stdout)` fail. The error is silently caught in createNodesV2, so external projects never register in the project graph. This single fix unblocks all 5 UAT tests.

Output: Patched extract.ts with env suppression, updated tests confirming the fix.
</objective>
