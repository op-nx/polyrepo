    Add a new test inside the existing `describe('polyrepo-status', ...)` block that exercises the full sync -> cache -> status-with-project-counts flow. Place it after the existing "should report unsynced repos" and "should register target on root project" tests.

    The nx.json config is already written in the `beforeAll` of the `polyrepo-status` describe block, pointing to `file:///repos/nx` (prebaked local repo in Docker image).

    **New test: "should show project counts after sync"**

    ```typescript
    it('should show project counts after sync', async () => {
      expect.assertions(2);

      // Run sync -- clones from file:///repos/nx to /workspace/.repos/nx/
      // and extracts graph cache
      const syncResult = await container.exec(
        ['npx', 'nx', 'polyrepo-sync'],
        { workingDir: '/workspace' },
      );

      if (syncResult.exitCode !== 0) {
        throw new Error(`Sync failed: ${syncResult.output}`);
      }

      // Run status -- should now show project counts from cached graph
      const statusResult = await container.exec(
        ['npx', 'nx', 'polyrepo-status'],
        { workingDir: '/workspace' },
      );

      expect(statusResult.stdout).toContain('projects');
      expect(statusResult.stdout).not.toContain('[not synced]');
    }, 120_000);
    ```

    Key details:
    - Use 120_000 ms timeout (120s) on this specific test because `nx polyrepo-sync` runs `nx graph --print` on the nrwl/nx repo inside the container, which can take 30-60s even with NX_DAEMON=false.
    - The `file:///repos/nx` repo is prebaked in the Docker image (shallow clone of nrwl/nx at master).
    - Assert stdout contains "projects" (the status output shows "N projects" for synced repos).
    - Assert stdout does NOT contain "[not synced]" (the repo should now be synced).
    - Throw on non-zero exit code from sync to get clear failure messages.
