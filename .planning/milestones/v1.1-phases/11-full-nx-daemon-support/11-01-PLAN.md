<objective>
Refactor the monolithic single-file graph cache into a per-repo cache architecture with three layers (global in-memory gate, per-repo disk, per-repo extraction), add exponential backoff with hash-change reset for extraction failures, and clean up the old cache file.

Purpose: Eliminate the cold-start `NX_DAEMON=false` requirement by enabling selective per-repo cache invalidation and disk recovery. Under the daemon, only stale repos re-extract while unchanged repos return instantly from memory. Backoff prevents repeated 30-60s extraction penalties on rapid Nx command sequences.

Output: Refactored `cache.ts` with per-repo cache files at `.repos/<alias>/.polyrepo-graph-cache.json`, comprehensive unit tests, and exported `computeRepoHash`/`writePerRepoCache` for Plan 02 (sync pre-caching).
</objective>

Where `RepoGraphData` is the per-repo report type: `{ nodes: Record<string, TransformedNode>; dependencies: Array<...> }`.

Key functions to implement:

- `getPerRepoCachePath(workspaceRoot, alias)` -- returns `.repos/<alias>/.polyrepo-graph-cache.json`
- `tryReadPerRepoCache(workspaceRoot, alias)` -- reads per-repo cache file, returns `{ hash, report }` or undefined
- `writePerRepoCache(workspaceRoot, alias, hash, report)` -- writes per-repo cache file (exported for sync executor)
- `computeRepoHash(reposConfigHash, alias, repoPath)` -- computes per-repo hash from `hashArray([reposConfigHash, alias, headSha, dirtyFiles])` (exported for sync executor)
- `computeGlobalHash(config, workspaceRoot, reposConfigHash)` -- combines all per-repo hashes into one global hash
- `shouldSkipExtraction(alias, currentHash)` -- checks backoff state, returns true if within cooldown
- `recordFailure(alias, currentHash)` -- records extraction failure with timestamp
- `logExtractionFailure(alias, error)` -- logs actionable warning with 4 troubleshooting steps
- `cleanupOldCache(workspaceRoot)` -- deletes old monolithic cache if it exists (one-time)
- `populateGraphReport(config, workspaceRoot, reposConfigHash)` -- the main function, refactored

The `populateGraphReport` flow:

1. Clean up old cache (one-time flag)
2. Compute global hash from all per-repo hashes
3. If global hash matches, return assembled report from `perRepoCache` Map
4. For each synced repo entry:
   a. Compute per-repo hash
   b. Check in-memory per-repo cache (Map lookup)
   c. Check disk cache (`tryReadPerRepoCache`)
   d. Check backoff (`shouldSkipExtraction`) -- if skipped, log warning, continue without this repo
   e. Extract + transform, write per-repo cache
   f. On failure: `recordFailure`, `logExtractionFailure`, continue without this repo
5. Update global hash
6. Assemble and return report

Rename the `pluginOptionsHash` parameter to `reposConfigHash` throughout.
