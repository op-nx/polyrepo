  <action>
Create a shell script `create-test-repo.sh` that builds a tiny synthetic Nx workspace at `/repos/test-repo`:

1. Run `npx --yes create-nx-workspace@${NX_VERSION} test-repo --preset=apps --interactive=false --nxCloud=skip` in a temp directory
2. Generate 2 simple libraries inside it using `npx nx g @nx/js:lib lib-a --directory=libs/lib-a --interactive=false` and `npx nx g @nx/js:lib lib-b --directory=libs/lib-b --interactive=false` (use @nx/js since it is lightweight -- do NOT use @nx/react or @nx/node which pull in heavy deps). If @nx/js:lib generator is not available at this Nx version, create the libs manually: make `libs/lib-a/` and `libs/lib-b/` directories each with a `project.json` containing `{ "name": "lib-a", "sourceRoot": "libs/lib-a/src" }` (and lib-b respectively), and a `src/index.ts` with `export const greeting = 'hello';`.
3. Run `npm install` to ensure node_modules are populated (the workspace uses npm by default from create-nx-workspace --preset=apps)
4. Initialize as a git repo: `git init && git add -A && git commit -m "init"`
5. Move the result to `/repos/test-repo`

The script should use `set -euo pipefail` and echo progress messages (no emojis, use `[INFO]` prefix).

Update the Dockerfile:

- Remove the `ARG NX_REF` and `RUN git clone --depth 1 --branch ${NX_REF} https://github.com/nrwl/nx.git /repos/nx` lines
- Add `COPY create-test-repo.sh /tmp/create-test-repo.sh`
- Add `RUN chmod +x /tmp/create-test-repo.sh && /tmp/create-test-repo.sh` after the create-nx-workspace step (so NX_VERSION ARG is available)
- The script should receive NX_VERSION via the ARG already set in the Dockerfile (use `ARG NX_VERSION` again before the RUN if needed, or pass it as env)
- Keep the existing create-nx-workspace step that creates `/workspace` -- that is the HOST workspace. The test-repo at `/repos/test-repo` is the EXTERNAL repo that gets synced into it.

Important: The test-repo must have its own `node_modules/` with Nx installed so that `extractGraphFromRepo` can find `node_modules/.bin/nx` when it runs `nx graph --print` inside the cloned repo. The `npm install` in step 3 ensures this. However, since the sync executor's `tryInstallDeps` will also run `npm install` after cloning, having pre-installed deps means the lockfile hash check (`needsInstall`) will detect no change and skip the install. This is the key performance optimization.

Wait -- the sync executor clones FROM `/repos/test-repo` TO `/workspace/.repos/test-repo/`. The clone is a fresh `git clone`, so node_modules will NOT be in the clone (git does not track node_modules). The sync executor will then run `npm install` in the cloned copy. Since this is a tiny workspace (2-3 projects), npm install should complete in under 10s. Then `nx graph --print` on 2-3 projects takes ~2-3s. Total well under 120s.

So the key insight is: we do NOT need to prebake node_modules in the git repo. We just need the repo to be tiny so that install + graph extraction is fast. The prebaked `/repos/test-repo` with node_modules is for the `git clone --local` source only -- git will clone just the tracked files.
