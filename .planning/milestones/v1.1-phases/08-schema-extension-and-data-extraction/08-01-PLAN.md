    1. Remove `_` prefix from `_workspaceRoot` parameter (rename to `workspaceRoot`).
    2. Add imports: `import { readFileSync } from 'node:fs';` and `import { join } from 'node:path';` (join may already be imported -- check).
    3. Inside the node iteration loop, BEFORE root rewriting, extract the package name from typed metadata:
    ```typescript
    const packageName = node.data.metadata?.js?.packageName;
    ```
    This is now type-safe because Task 1 refined the metadata schema.

    4. Read dependency lists from package.json on disk. Construct the path using `workspaceRoot` + `.repos/<alias>` + `node.data.root` (original root, before rewriting):
    ```typescript
    const repoBasePath = join(workspaceRoot, '.repos', repoAlias);
    const pkgJsonPath = join(repoBasePath, node.data.root, 'package.json');
    let dependencies: string[] | undefined;
    let devDependencies: string[] | undefined;
    let peerDependencies: string[] | undefined;

    try {
      const raw = JSON.parse(readFileSync(pkgJsonPath, 'utf-8'));

      if (raw.dependencies && typeof raw.dependencies === 'object') {
        dependencies = Object.keys(raw.dependencies);
      }

      if (raw.devDependencies && typeof raw.devDependencies === 'object') {
        devDependencies = Object.keys(raw.devDependencies);
      }

      if (raw.peerDependencies && typeof raw.peerDependencies === 'object') {
        peerDependencies = Object.keys(raw.peerDependencies);
      }
    } catch {
      // No package.json or parse error -- silent skip per locked decision
    }
    ```

    5. Add the new fields to the node assignment:
    ```typescript
    nodes[namespacedName] = {
      // ... existing fields ...
      packageName: typeof packageName === 'string' ? packageName : undefined,
      dependencies,
      devDependencies,
      peerDependencies,
    };
    ```

    **IMPORTANT path construction note:** Use `node.data.root` (the ORIGINAL root from the graph, e.g., `libs/my-lib`) NOT the rewritten `hostRoot` (`.repos/repo-b/libs/my-lib`). The `repoBasePath` already includes `.repos/<alias>`. This avoids the double-path pitfall from RESEARCH.md Pitfall 3.
