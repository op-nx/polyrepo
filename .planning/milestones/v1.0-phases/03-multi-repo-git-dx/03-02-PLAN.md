<task type="auto">
  <name>Task 1: Rewrite status executor with aligned columns, auto-fetch, warnings, project counts</name>
  <files>packages/op-nx-polyrepo/src/lib/executors/status/executor.ts</files>
  <action>
    Rewrite the status executor completely. The new executor flow:

    1. **Config loading** (same pattern as current): read nx.json, find plugin entry, validateConfig, normalizeRepos.

    2. **Determine repo paths and states**: For each NormalizedRepoEntry, compute the repo path (`.repos/<alias>` for remote, `entry.path` for local) and call `detectRepoState`. Separate into synced entries (state !== 'not-synced') and unsynced entries.

    3. **Auto-fetch (parallel)**: For synced entries, run `gitFetch(repoPath)` in parallel via `Promise.allSettled`. Log warnings for fetch failures but do not abort -- proceed with stale remote refs. Import `gitFetch` from `../../git/commands`.

    4. **Gather state per repo (parallel)**: For each synced entry, in parallel gather:
       - `getCurrentBranch(repoPath)` -- null means detached HEAD
       - `getCurrentRef(repoPath)` -- tag or short SHA (for detached HEAD display)
       - `getWorkingTreeState(repoPath)` -- file category counts
       - `getAheadBehind(repoPath)` -- only if NOT tag-pinned AND NOT detached HEAD. Tag detection: reuse the `isTagRef` pattern from sync executor (`/^v?\d+\.\d+/`). If branch is null (detached) AND getCurrentRef returns a tag, treat as tag-pinned.
       - `getProjectCount(workspaceRoot, alias)` -- read graph cache

    5. **Read graph cache for project counts**: Create a local helper `getProjectCount(workspaceRoot: string, alias: string): number | null` that:
       - Reads `.repos/.polyrepo-graph-cache.json` using `readJsonFile` from `@nx/devkit`
       - Looks up `cache.report.repos[alias]`
       - Returns `Object.keys(repoReport.nodes).length` if found, `null` if not found or on any error
       - This is called once per repo -- could be optimized to read cache once, but simplicity is preferred

    6. **Build row data**: For each entry (synced and unsynced), build a row for the aligned table:
       - Column 1: alias (left-aligned)
       - Column 2: branch/ref display. For synced: branch name, or tag name, or `(detached)`. For unsynced: `[not synced]`
       - Column 3: ahead/behind. For branch-tracked repos: `+N -N`. For tag-pinned or detached: empty. For unsynced: empty. If getAheadBehind returned null: empty (no `?` -- the auto-fetch should have made this accurate for non-shallow repos; for shallow repos the column is simply blank)
       - Column 4: dirty summary. For clean repos: `clean`. For dirty: space-separated non-zero counts like `3M 1A 2D 1??`. For unsynced: empty
       - Column 5: project count. `N projects` or `? projects`. For unsynced: `? projects`
       - Column 6: warnings (joined). Multiple warnings possible per repo

    7. **Warning logic** (per locked decisions):
       - `[WARN: dirty, sync may fail]` -- if modified + staged + deleted + untracked > 0
       - `[WARN: detached HEAD]` -- if getCurrentBranch returned null AND not tag-pinned
       - `[WARN: merge conflicts]` -- if conflicts > 0
       - `[WARN: drift]` -- if entry has a configured ref AND current branch/tag does not match it. Also show `(expected <ref>)` next to the branch/ref display in column 2

    8. **Format and output**: Use `formatAlignedTable` from `../../format/table` to align all rows. Output each line via `logger.info`. Then output the summary line: `N configured, M synced, K not synced`. Then output the legend (always, per locked decision):
       ```
       Legend:
         M  = modified files
         A  = staged/added files
         D  = deleted files
         ?? = untracked files
         +N = commits ahead of remote
         -N = commits behind remote
         ?  = graph not yet extracted (run any nx command to trigger)
       ```

    9. **Return**: Always `{ success: true }` per locked decision. Exit 1 only on executor-level failure (config read, git binary missing), which is handled by the try/catch wrapping the entire executor body.

    Keep `StatusExecutorOptions` as `Record<string, never>` (no new options -- skip-fetch is deferred).

    The `isTagRef` helper should be a local function (same regex `/^v?\d+\.\d+/` as sync executor). Do NOT import from sync -- avoid coupling executors.

    For the dirty summary formatting, build a string like `3M 1A 2D 1??` by conditionally appending non-zero categories. If all zero, use `clean`.
  </action>

  </verify>
