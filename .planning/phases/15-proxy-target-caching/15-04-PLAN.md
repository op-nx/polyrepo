<objective>
Include the plugin version (from package.json) in the graph disk cache key so that upgrading the plugin invalidates stale cached target configurations.

Purpose: After the Plan 01 change from `cache: false` to `cache: true` on proxy targets, the graph disk cache at `.repos/<alias>/.polyrepo-graph-cache.json` served stale `cache: false` values because the cache key (`computeRepoHash`) only included HEAD SHA + dirty files + repos config hash — not the plugin version. Deleting the disk cache manually fixed it, but this should happen automatically on plugin upgrade.

Output: Updated `computeRepoHash` that includes plugin version, ensuring cache auto-invalidates on upgrade.

    1. **Read the plugin version at module load time** in `cache.ts`:

       Add a module-level constant that captures the plugin version once:
       ```typescript
       import { readJsonFile } from '@nx/devkit';
       import { join } from 'node:path';

       // Read plugin version at module load for cache key invalidation.
       // When the plugin is upgraded, the version changes, invalidating all
       // disk caches — ensuring stale target configs (e.g., old cache: false)
       // are never served after an upgrade.
       const PLUGIN_VERSION: string = (() => {
         try {
           const pkg = readJsonFile<{ version: string }>(
             join(__dirname, '..', '..', '..', 'package.json'),
           );
           return pkg.version;
         } catch {
           // Fallback: if package.json is unreadable, use a constant that
           // changes on every process start (forces re-extraction).
           return Date.now().toString();
         }
       })();
       ```

       The `__dirname` in the compiled output is `dist/packages/op-nx-polyrepo/src/lib/graph/`, so `../../../package.json` resolves to `dist/packages/op-nx-polyrepo/package.json`. In the source (when loaded from TS by Nx), `__dirname` is `packages/op-nx-polyrepo/src/lib/graph/`, so `../../../package.json` resolves to `packages/op-nx-polyrepo/package.json`. Both are correct.

       **However**, the Nx TS transpiler may not define `__dirname` correctly for ESM-style modules. Safer approach — use `require.resolve` or a known import path:
       ```typescript
       // Safer: resolve relative to the module's own location
       import { fileURLToPath } from 'node:url';
       ```

       Actually, since the project uses `module: "node16"` which compiles to CJS, `__dirname` is available natively. Use the `__dirname` approach.

       **Alternative approach (even simpler):** Import the version from the package.json directly:
       ```typescript
       // At the top of cache.ts, add:
       import { join, dirname } from 'node:path';

       // Module-level constant — read once at load time
       let pluginVersion: string;
       try {
         // eslint-disable-next-line @typescript-eslint/no-require-imports
         const pkg = require('../../../package.json') as { version: string };
         pluginVersion = pkg.version;
       } catch {
         pluginVersion = `dev-${Date.now()}`;
       }
       ```

       Choose whichever approach works with the project's TS/lint configuration. The key requirement is: the version string is included in the hash and changes on plugin upgrade.
