<objective>
Add pre-caching of graph data during `polyrepo-sync` so the first daemon invocation after sync hits a warm per-repo disk cache instead of triggering expensive extraction.

Purpose: Eliminate the cold-start problem. After `nx polyrepo-sync`, the daemon's first graph computation reads from disk (~50-200ms) instead of spawning child `nx graph --print` processes (~4-60s per repo). This is the final piece that makes `NX_DAEMON=true` the reliable default.

Output: Modified sync executor that extracts and caches graph data after each repo's install completes, with warn-and-continue on failure. Unit tests verifying pre-cache behavior.
</objective>

Add a `preCacheGraph` helper function (private to the module):

```typescript
async function preCacheGraph(
  repoPath: string,
  alias: string,
  workspaceRoot: string,
  reposConfigHash: string,
): Promise<void> {
  logger.info(`Extracting graph for ${alias}...`);

  try {
    const rawGraph = await extractGraphFromRepo(repoPath);
    const transformed = transformGraphForRepo(alias, rawGraph, workspaceRoot);
    const hash = await computeRepoHash(reposConfigHash, alias, repoPath);

    writePerRepoCache(workspaceRoot, alias, hash, {
      nodes: transformed.nodes,
      dependencies: transformed.dependencies,
    });

    const projectCount = Object.keys(transformed.nodes).length;
    logger.info(`Cached graph for ${alias} (${String(projectCount)} projects)`);
  } catch (error) {
    logger.warn(
      `Failed to pre-cache graph for ${alias}: ${error instanceof Error ? error.message : String(error)}`,
    );
    logger.warn('Plugin will extract on next Nx command.');
  }
}
```
