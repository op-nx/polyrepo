Run lint and typecheck on production code, then fix ALL violations. Expected violation categories:

1. **`noUncheckedIndexedAccess`** -- Array/Record indexing now returns `T | undefined`. Add null checks or use `for...of Object.entries()` destructuring where safe. Per Pitfall 5: destructuring in for-of loops gives `T` not `T | undefined`. Prefer `if` guards over `!` assertions.

2. **`exactOptionalPropertyTypes`** (if enabled in Plan 01) -- May cause issues with @nx/devkit types. If so, Plan 01 already removed it. If enabled, fix any assignments of `undefined` to optional properties by using `delete` or conditional spreading.

3. **`@typescript-eslint/explicit-function-return-type`** -- Add explicit return types to all exported and non-exported functions in production code. Arrow functions passed as arguments and typed function expressions are exempt (allowExpressions, allowTypedFunctionExpressions).

4. **`@typescript-eslint/consistent-type-imports`** -- Change `import { Foo } from` to `import { type Foo } from` or `import type { Foo } from` for type-only imports. The autofix should handle most of these: run `npm exec nx lint @op-nx/polyrepo -- --fix` first.

5. **`@typescript-eslint/consistent-type-exports`** -- Add `type` qualifier to type-only re-exports. Autofix available.

6. **`@typescript-eslint/consistent-type-definitions`** -- Convert `type Foo = { ... }` to `interface Foo { ... }` for object shapes. Keep `type` for unions, intersections, and mapped types.

7. **`no-floating-promises`** -- Ensure all Promise returns are awaited or explicitly voided. Check executor code carefully -- it's heavily async.

8. **`no-unsafe-*` family** -- Fix any unsafe any usage (unlikely in production code since there are no explicit `any` types, but may appear from untyped library returns).

9. **`@typescript-eslint/no-non-null-assertion`** -- Replace any `!` assertions with proper null checks.
