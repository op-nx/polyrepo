## Tech Debt

### Phase 03 -- Multi-Repo Git DX

1. **Hardcoded cache filename** -- `status/executor.ts` line 44 contains the literal string `'.polyrepo-graph-cache.json'`, duplicating the `CACHE_FILENAME` constant defined as module-private in `cache.ts`. If the filename changes in `cache.ts`, the status executor's project-count display silently degrades to `'? projects'` rather than failing. Both files should reference a shared exported constant.

2. **Duplicated config reading** -- Both `syncExecutor` (lines 430-443) and `statusExecutor` (lines 98-112) contain identical boilerplate to `readFileSync(nxJsonPath)` and extract plugin options from the nx.json plugins array. This config-extraction logic could be shared via a utility function.

### Total: 2 items across 1 phase
