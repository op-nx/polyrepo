  <name>Task 1: Add --dry-run option and aligned summary table to sync executor</name>
  <files>packages/op-nx-polyrepo/src/lib/executors/sync/executor.ts, packages/op-nx-polyrepo/src/lib/executors/sync/schema.json</files>
  <action>
    **schema.json changes:** Add `dryRun` boolean property (default false) to the existing schema:
    ```json
    "dryRun": {
      "type": "boolean",
      "default": false,
      "description": "Show what sync would do without executing"
    }
    ```

    **SyncExecutorOptions changes:** Add `dryRun?: boolean` to the interface.

    **Dry-run mode** (when `options.dryRun` is true):

    For each entry, determine what sync WOULD do:
    1. Compute `detectRepoState` to know if clone or update
    2. For synced repos, call `getWorkingTreeState` to check if dirty (import from `../../git/detect`)
    3. Determine action:
       - `not-synced` + remote entry -> `would clone`
       - `not-synced` + local entry -> `would skip (path not found)`
       - cloned/referenced + tag ref (isTagRef) -> `would fetch tag <ref>`
       - cloned/referenced + branch -> `would ${strategy}` (e.g., "would pull", "would fetch", "would rebase")
    4. If dirty (modified+staged+deleted+untracked > 0), add warning: `[WARN: dirty, may fail]`
    5. Output dry-run results using `formatAlignedTable` with columns: alias, action, warning
    6. Output summary: `Dry run: N would sync, M would skip`
    7. Return `{ success: true }` (dry-run never fails)

    **Summary table** (for normal sync -- not dry-run):

    After the existing `Promise.allSettled` loop, collect per-repo results into an array of `{ alias: string, action: string, outcome: string }`:
    - action: `cloned`, `updated`, `fetched tag`, `pull`, `skip` (matches existing logger.info messages)
    - outcome: `[OK]` on fulfilled, `[ERROR] <message>` on rejected

    The result collection logic:
    - Track what action each repo took. Modify `syncRepo` to return a result descriptor instead of void, OR collect results from the existing logger.info calls by tracking state before and after.
    - Simpler approach: have `syncRepo` return `{ alias: string, action: string }` on success. On failure, the Promise.allSettled rejection reason + the entry provides the context.

    Change `syncRepo` return type from `Promise<void>` to `Promise<{ action: string }>`. Return `{ action: 'cloned' }`, `{ action: 'updated' }`, `{ action: 'fetched tag' }`, etc. For local skip, return `{ action: 'skipped' }`.

    After `Promise.allSettled`, build rows for the summary table:
    ```
    Results:
    repo-a     cloned    [OK]
    repo-b     updated   [OK]
    repo-ccc   pull      [ERROR] auth denied
    ```

    Use `formatAlignedTable` for alignment. Output via `logger.info('')` (blank line before), then `logger.info('Results:')`, then each formatted row via `logger.info`.

    Keep the existing summary line after the table: `N synced, M failed`.

    **Important:** Keep the streaming progress lines (Cloning..., Updating..., Done:) as they are. The summary table is ADDED at the end, it does not replace them. Per CONTEXT.md: "sync keeps streaming progress lines during execution, then adds an aligned results table at the end".

    The `isTagRef` function is already defined locally in sync/executor.ts -- keep it as is.

    Import `formatAlignedTable` and `ColumnDef` from `../../format/table`. Import `getWorkingTreeState` from `../../git/detect` (only needed in dry-run path).
