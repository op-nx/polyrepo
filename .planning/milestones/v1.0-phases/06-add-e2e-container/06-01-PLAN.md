Create `packages/op-nx-polyrepo-e2e/src/setup/global-setup.ts` implementing the full testcontainers lifecycle:

1. **Import** `GenericContainer`, `Network` from `testcontainers`, `releaseVersion`/`releasePublish` from `nx/release`, `TestProject` from `vitest/node`. Side-effect import `./provided-context.ts` for type augmentation.

2. **Export default async function** `setup(project: TestProject)`:

   a. **Build Docker image** â€” Use `GenericContainer.fromDockerfile()` pointing at `packages/op-nx-polyrepo-e2e/docker/` to build the prebaked workspace image. If `fromDockerfile()` does not support build args well, fall back to `execSync('docker build -t op-nx-e2e-workspace:latest packages/op-nx-polyrepo-e2e/docker/')` and then use `new GenericContainer('op-nx-e2e-workspace:latest')`. Use Claude's discretion on which approach works better.

   b. **Create shared network** â€” `const network = await new Network().start();`

   c. **Start Verdaccio** â€” `hertzg/verdaccio` image on the shared network:
   - `.withNetwork(network)`
   - `.withNetworkAliases('verdaccio')`
   - `.withExposedPorts(4873)`
   - `.withName('op-nx-polyrepo-e2e-verdaccio')`
   - Use `.withWaitStrategy(Wait.forListeningPorts())` for readiness detection
   - Get mapped port: `verdaccio.getMappedPort(4873)`

   d. **Publish plugin to Verdaccio** â€” Set `process.env['npm_config_registry']` to `http://localhost:${mappedPort}`, then call `releaseVersion()` and `releasePublish()` with the same arguments as current `start-local-registry.ts`. Clean up the env var after publish.

   e. **Start workspace container** from the built image:
   - `.withNetwork(network)`
   - `.withName('op-nx-polyrepo-e2e-workspace')`
   - `.withCommand(['sleep', 'infinity'])` (if not already CMD in Dockerfile â€” the container needs to stay alive)

   f. **Install plugin in container** â€” `await workspace.exec(['npm', 'install', '-D', '@op-nx/polyrepo@e2e', '--registry', 'http://verdaccio:4873'], { workingDir: '/workspace' })`. Check exitCode and throw if non-zero.

   g. **Commit snapshot** â€” `await workspace.commit({ repo: 'op-nx-e2e-snapshot', tag: 'latest', deleteOnExit: true })`. This captures the filesystem state with plugin installed.

   h. **Provide to test files** â€” `project.provide('snapshotImage', snapshotImage)` and `project.provide('networkName', network.getName())`.

   i. **Stop setup workspace** â€” `await workspace.stop()` (tests use snapshot, not this container).

   j. **Return teardown function** that stops Verdaccio and the network:

   ```typescript
   return async function teardown() {
     await verdaccio.stop();
     await network.stop();
   };
   ```

3. **Error handling**: Wrap the entire setup in try/catch. On failure, attempt to stop any started containers/network before re-throwing. Log progress with console.log for each major step (building image, starting Verdaccio, publishing, installing, committing snapshot).

4. **TESTCONTAINERS_RYUK_DISABLED**: Do NOT set this in the setup code â€” it is only for CI (set via env var in CI workflow). Locally, Ryuk provides valuable auto-cleanup.
