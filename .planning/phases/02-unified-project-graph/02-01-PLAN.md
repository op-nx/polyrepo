    **1. Create `packages/op-nx-polyrepo/src/lib/graph/types.ts`:**

    Define TypeScript interfaces matching the `nx graph --print` JSON output structure (verified from nrwl/nx source). The key types are:

    - `ExternalGraphJson` -- raw output from `nx graph --print`: `{ graph: { nodes: Record<string, ExternalProjectNode>, dependencies: Record<string, ExternalDependency[]> } }`. `ExternalProjectNode` has `name: string`, `type: string`, `data: { root: string, targets?: Record<string, TargetConfiguration>, tags?: string[], metadata?: Record<string, unknown>, sourceRoot?: string, projectType?: string }`. `ExternalDependency` has `source: string`, `target: string`, `type: string`.
    - `TransformedNode` -- namespaced project node: `{ name: string, root: string, projectType?: string, sourceRoot?: string, targets: Record<string, TargetConfiguration>, tags: string[], metadata?: Record<string, unknown> }`.
    - `PolyrepoGraphReport` -- aggregated report across all repos: `{ repos: Record<string, { nodes: Record<string, TransformedNode>, dependencies: Array<{ source: string, target: string, type: string }> }> }`.
    - Import `TargetConfiguration` from `@nx/devkit`.

    **2. Extend `packages/op-nx-polyrepo/src/lib/git/detect.ts`:**

    Add two new exported async functions using the existing `execGitOutput` pattern (currently private in detect.ts -- refactor to export it, OR duplicate the pattern inline):

    - `getHeadSha(cwd: string): Promise<string>` -- runs `git rev-parse HEAD` and returns trimmed output.
    - `getDirtyFiles(cwd: string): Promise<string>` -- runs `git diff --name-only HEAD` and returns trimmed output (empty string = clean).

    Follow the existing `execGitOutput` pattern already in detect.ts (promisified execFile). The function is already there as a private helper -- either export it or use it internally for the new functions.

    **3. Create `packages/op-nx-polyrepo/src/lib/git/normalize-url.ts`:**

    Implement `normalizeGitUrl(url: string): string` following the exact algorithm from 02-RESEARCH.md:
    - Trim whitespace
    - Strip trailing `.git`
    - Normalize `git@host:org/repo` SSH syntax to `https://host/org/repo`
    - Normalize `ssh://git@host/path` to `https://host/path`
    - Replace `git://` with `https://`
    - Lowercase the hostname portion of https:// URLs using `new URL()`
    - Strip trailing `/` from final result
    - Wrap URL parsing in try/catch -- non-parseable strings returned as-is (for local paths)

    **4. Write tests** following existing patterns (vi.mock for node:child_process in detect tests, pure function tests for normalize-url).

    - Rejects config where two repos have the same normalized URL (e.g., SSH and HTTPS forms of same repo)
    - Rejects config where two repos have same URL but one has .git suffix
    - Accepts config where repos have genuinely different URLs
    - For path-based repos: shells out to `git remote get-url origin` for URL comparison, falls back to resolved absolute path if remote lookup fails
    - Error message identifies the conflicting aliases

    Sync dep install tests (extend executor.spec.ts):
    - After cloning a remote repo, runs `npm install` (when package-lock.json detected)
    - After cloning a remote repo, runs `pnpm install` (when pnpm-lock.yaml detected)
    - After cloning a remote repo, runs `yarn` (when yarn.lock detected)
    - After pulling an existing remote repo, runs install command
    - For local path repos that are updated, runs install command
    - Install failure logs warning but does not fail the sync (degraded mode -- repo is synced but deps may be stale)
  </behavior>
  <action>
    **1. Extend `packages/op-nx-polyrepo/src/lib/config/schema.ts`:**

    Add a `.refine()` to the `polyrepoConfigSchema` (chain after the existing `.refine()` for non-empty repos) that detects duplicate repo URLs:

    - Import `normalizeGitUrl` from `../git/normalize-url`.
    - For each repo entry, extract the URL: string entries that match gitUrlPattern get the string itself, object entries with `url` get `entry.url`, path entries get the path string.
    - Normalize each URL using `normalizeGitUrl`.
    - For path-based entries: normalization uses the path as-is (normalizeGitUrl returns non-URL strings unchanged). Note: shelling out to `git remote get-url origin` during zod validation is synchronous context -- instead, use `resolved absolute path` comparison via `path.resolve()`. The async git remote lookup is NOT feasible in zod's synchronous refine. Use `path.resolve()` to normalize relative paths for comparison.
    - Build a map of normalized URL -> alias[]. If any URL maps to 2+ aliases, return false with a message listing the conflicting aliases.
    - Use zod `path` to indicate which field has the issue.

    - Check for `pnpm-lock.yaml` -> return `'pnpm'`
    - Check for `yarn.lock` -> return `'yarn'`
    - Default (including `package-lock.json`) -> return `'npm'`
    - Use `existsSync` from `node:fs`.

    Create a helper `installDeps(repoPath: string): Promise<void>`:
    - Detect package manager using `detectPackageManager`.
    - Map to install command: `npm` -> `['npm', 'install']`, `pnpm` -> `['pnpm', 'install']`, `yarn` -> `['yarn']`.

The repo is synced; deps are a best-effort step. Message: `"Failed to install dependencies for <alias>: <error>. Run install manually in .repos/<alias>/"`

    For local repos, the `repoPath` is `entry.path` (already resolved). For remote repos, `repoPath` is `join(workspaceRoot, '.repos', entry.alias)`.
