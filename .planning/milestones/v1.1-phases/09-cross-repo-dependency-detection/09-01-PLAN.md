    Implementation notes (based on locked decisions in CONTEXT.md + pitfall guidance from RESEARCH.md):

    1. Build lookup map:
       - Iterate all repos → nodes in report.repos. For each TransformedNode with a non-undefined `packageName`, insert `packageName → node.name` (if key not already in map).
       - Then iterate `context.projects`. For each project whose `metadata?.['js']?.['packageName']` is a string, insert if key not already in map.
       - Use `Map.has()` guard: `if (!map.has(pkgName)) { map.set(pkgName, projectName); }` — external TransformedNode entries inserted first so they take precedence over host.

    2. Build reverse-lookup for repo membership:
       - Build a `Map<string, string>` from `namespacedProjectName → repoAlias` (repo-a/my-lib → repo-a, or 'host' for host projects).
       - For external nodes: extract alias from the namespaced name's prefix (everything before the first `/`).
       - For host projects: use a sentinel string like `'__host__'`.

    3. Emit edges (dep-list scan):
       - Collect edges into a `Set<string>` of `"source::target"` keys to deduplicate.
       - Iterate all external nodes (across all repos). For each, scan `dependencies`, `devDependencies`, `peerDependencies`. For each dep name found in lookup map, check cross-repo guard (source repo prefix !== target repo prefix). If cross-repo, add edge.
       - Iterate host projects from `context.projects`. Read `package.json` from `join(context.workspaceRoot, projectConfig.root, 'package.json')` with silent `try/catch`. Parse, extract dep arrays. Same scan and cross-repo guard. Use `'__host__'` as the source's repo sentinel.
       - `sourceFile` for external node: the `node.root` (already `.repos/<alias>/apps/my-app`) + `/package.json`, normalized to forward slashes.
       - `sourceFile` for host node: `join(projectConfig.root, 'package.json')` normalized to forward slashes (relative from workspaceRoot).

    4. No cycle detection — pass through to Nx. No OVRD/tsconfig logic yet (Plan 02).

    Type safety rules (project-enforced, see SKILL.md):
    - No `as` casts (except `as const`), no `any`, no `!` non-null assertions
    - Use `import type` for type-only imports
    - Exported function must declare return type explicitly
    - Index access via bracket notation on index signatures
    - Use `isRecord()` pattern (already in transform.ts) for narrowing `unknown`
    - Use `z.object().loose()` (not `passthrough()`) for Zod schemas
