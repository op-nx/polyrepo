  <behavior>
    executor.spec.ts (mock config reading, detect, and git functions):
    - shows 'cloned' state with path for synced remote repos
    - shows 'referenced' state with path for local repos that exist
    - shows 'not synced' state with URL for remote repos not yet cloned
    - shows current branch for synced repos
    - shows configured ref alongside current branch
    - shows [DRIFT] marker when current branch differs from configured ref
    - does not show [DRIFT] when configured ref is undefined (default branch)
    - does not show [DRIFT] when current branch matches configured ref
    - always returns { success: true } (informational command, never fails)
    - handles errors in individual repo detection gracefully (logs warning, continues)
  </behavior>
  <action>
    Create `packages/nx-openpolyrepo/src/lib/executors/status/schema.json`:
    ```json
    {
      "$schema": "https://json-schema.org/schema",
      "type": "object",
      "properties": {},
      "additionalProperties": false
    }
    ```

    Create `packages/nx-openpolyrepo/src/lib/executors/status/executor.ts`:

    1. Define `StatusExecutorOptions` as empty interface `{}`.

    2. Read and validate plugin config from nx.json (same pattern as sync executor -- `readNxJson`, find plugin entry, `validateConfig`, `normalizeRepos`).

    3. For each normalized repo entry:
       - Call `detectRepoState(alias, entry, root)` to get state and path
       - If state is 'cloned' or 'referenced': get current branch/ref via `getCurrentBranch(path)` / `getCurrentRef(path)`
       - Detect drift: if entry has `ref` configured AND current branch differs from configured ref, mark as `[DRIFT]`

    4. Format output per CONTEXT.md spec:
       ```
         repo-a: cloned (.repos/repo-a/)
           branch: main (configured: default)
         repo-b: referenced (D:/projects/repo-b)
           branch: develop
         repo-c: cloned (.repos/repo-c/)
           branch: feature-x (configured: develop) [DRIFT]
         repo-d: not synced
           url: git@github.com:org/repo-d.git
       ```
       Use `logger.info` for output lines.

    5. Always return `{ success: true }` -- status is informational.

    6. Wrap individual repo detection in try/catch -- if detection fails for one repo, log warning and continue to next.

    7. Export as default.

    Write tests FIRST (red), then implement (green). Mock `readNxJson`, `detectRepoState`, `getCurrentBranch`, `getCurrentRef`.
